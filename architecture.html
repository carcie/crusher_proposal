Arquitectura de referencia.

Nuestra arquitectura separa la capa de detección física de la capa de gestión de datos. Esto garantiza que una actualización de sensores o un cambio de software no comprometa el ecosistema de monitoreo.

01. Adquisición de Datos de Alta Fidelidad

La Capa Física: Detección de Alta Resistencia en Entornos Agresivos

Chancador: Análisis Térmico y de Vibración

Utilizamos acelerómetros MEMS de 3 ejes (rango de frecuencia: 2 Hz a 5.6 kHz) para capturar un perfil de vibración integral. El sistema monitorea la v‑RMS (para rastrear fatiga y resonancia estructural) y la a‑Peak (para detectar impactos de alta energía y degradación de pistas de rodamiento). Todas las mediciones se comparan con los estándares ISO 10816‑3, proporcionando un rango de medición antes de que se generen fallas estructurales.

Correas Transportadoras: Salud Molecular del Lubricante

En lugar de depender de cambios de aceite por intervalos fijos, recomendamos emplear sensores electroquímicos de calidad de aceite. Estos sensores monitorean la constante dieléctrica del lubricante en tiempo real, detectando oxidación e ingreso de humedad con una sensibilidad de 0.01% (100 ppm). Esto permite un mantenimiento basado en condición, reduciendo significativamente el riesgo y extendiendo la vida útil del equipo.

02. Normalización de Señales EDGE

Señales en Terreno y la Lógica Digital.

Estándar IO‑Link (IEC 61131‑9)

La implementación de IO‑Link, permite el procesamiento de doble canal: los datos de proceso cíclicos se utilizan para el monitoreo en tiempo real, mientras que los datos de diagnóstico acíclicos proporcionan los metadatos necesarios para un análisis profundo de causa raíz. También permite la parametrización remota de sensores, facilitando ajustes a nivel de terreno sin intervención física.

Nodos Ethernet Industrial (IE‑Nodes)

Las señales se agregan en nodos robustos y multiprotocolo. Estos dispositivos admiten EtherNet/IP, PROFINET y Modbus TCP/IP simultáneamente. Esto permite que el hardware se comunique directamente con los PLC existentes, mientras transmite simultáneamente telemetría a nuestra capa de datos.

03. Agnosticismo de Protocolos y Soberanía de Datos

JSON Pipeline: Open Standards para la Interoperabilidad Completa

“Vamos más allá de las cajas negras propietarias hacia una red transparente y definida por las necesidades del negocio.”
  • Mapeo a JSON Estructurado – Usando motores de mapeo de alto rendimiento (como JSON‑SCADA), proponemos convertir datos industriales en cargas útiles JSON. Esto elimina costosas licencias propietarios y garantiza que los datos sean legibles de forma nativa por navegadores, bases de datos y otras herramientas del negocio.
  • Integración MQTT y API REST – Alarmas críticas y telemetría se distribuyen mediante una arquitectura Pub/Sub MQTT, diseñada para entrega de baja latencia incluso en operaciones ancho de banda limitado. Adicionalmente, los datos serán accesibles a través de API REST, asegurando que estos no queden nunca bajo control de un proveedor.

04. Loop de Análisis y Mantenimiento

SCADA Web

La capa HMI está construida sobre estándares web modernos, proporcionando visualización en tiempo real del estado de la maquinaria en cualquier dispositivo autorizado. Debido a que el sistema está construido sobre un motor basado en JSON, maneja miles de puntos de datos con latencia sub-milisegundo, ofreciendo capacidades granulares de análisis del los equipos.

Gestión Automatizada de Mantenimiento (CMMS)

El motor de análisis está integrado directamente con la lógica de mantenimiento CMMS. Cuando un sensor detecta un traspaso del umbral determinado, el sistema genera automáticamente una orden de trabajo preventiva. Adjunta los manuales técnicos relevantes, identifica las piezas de repuesto necesarias en el inventario y alerta al equipo de mantenimiento antes de que la falla escale a una avería.

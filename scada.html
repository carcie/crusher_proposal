Real‑time Visualization Core.
A portable, scalable SCADA/IoT foundation.

Built on mainstream open‑source IT tools, this layer provides a vendor‑independent, horizontally scalable platform for monitoring, control, and data integration – from a single machine to global clusters.

01. Real‑time Core & Persistence

MongoDB as the unified data backbone.

Document‑Oriented Real‑time Database

The platform leverages MongoDB as the single source of truth – serving as real‑time core, persistent configuration store, and Sequence of Events (SOE) historian. The NoSQL, schema‑less data model allows unlimited tags, servers, and users without predefined schemas, enabling rapid adaptation to evolving field requirements.

Change Streams & Asynchronous Events

Using MongoDB Change Streams, the system processes database events in real time. Every data insertion, update, or deletion triggers immediate, asynchronous reactions – from alarm generation to historian logging – without polling, ensuring sub‑second latency even under high throughput.

02. Web‑Native HMI & Configuration

Zero‑client visualization, accessible from any device.

HTML5 Responsive Interface

The human‑machine interface is built entirely on HTML5, JavaScript, and SVG. It runs natively in any modern browser – desktop, tablet, or mobile – without plugins. Internationalization (UTF‑8/I18N) is standard, supporting multilingual deployments across global sites.

Inkscape‑Based Synoptic Editor

Dynamic process graphics are created using Inkscape, a professional open‑source vector graphics editor. Engineers design SVG synoptics with familiar drawing tools, then link them to live data points via custom attributes – no proprietary drawing software required.

Web‑Based Configuration Manager

All system parameters – tags, drivers, alarms, users – are configured through a secure web interface. Changes are applied live, without restarting services, enabling dynamic adaptation to process modifications.

Role‑Based Access Control (RBAC)

Fine‑grained permissions control who can view, edit, or acknowledge alarms. RBAC integrates with existing LDAP/Active Directory or operates as a standalone authentication system, ensuring operational security.

03. Protocol Gateway & Edge Architecture

Unlimited connectivity, horizontally scalable.

  • Extensive Driver Library – Native support for IEC 60870‑5‑104, DNP3, Modbus (TCP/RTU), MQTT, OPC UA, Siemens S7, BACnet, IEC 61850 MMS/GOOSE, and many more. New drivers can be added in any language (Node.js, C#, Golang) by simply reading from and writing to MongoDB collections.
  • Lightweight Redundant Data Acquisition Nodes – Distributed collectors (e.g., Raspberry Pi) connect securely over TLS to the central database. Each node can handle multiple protocols, and nodes can be clustered for redundancy and load sharing. Horizontal scaling is achieved through MongoDB sharding and container orchestration (Docker, Kubernetes).
  • Edge Processing & Data Concentration – Nodes can perform local calculations, alarm filtering, and data aggregation before sending to the core, reducing bandwidth usage in remote or low‑connectivity pit environments.

04. Historian & Advanced Analytics

PostgreSQL/TimescaleDB Historian

High‑resolution time‑series data is stored in PostgreSQL with the TimescaleDB extension, optimized for fast inserts and complex temporal queries. This historian automatically aggregates raw telemetry into configurable intervals (minute, hour, day) for long‑term trend analysis.

Native Grafana Integration

The historian connects directly to Grafana, enabling instant creation of dashboards, alerts, and ad‑hoc queries. Mechanical engineers can visualize vibration spectra, temperature trends, and oil quality degradation without writing SQL. Grafana’s alerting engine integrates with the event viewer for automated notification escalation.

Big Data & ML Extensibility

Through the MongoDB Spark connector, the platform feeds directly into big data pipelines and machine learning frameworks. Predictive models for bearing failure or conveyor misalignment can be trained on the full historian dataset and deployed back to the edge.

Third‑Party Tool Integration

Standard connectors to Power BI, Metabase, Node‑RED, Alerta, and Kafka are available. The platform can also serve as a data concentrator for ERP/MES systems, bridging real‑time operations with enterprise planning.

Visualización de Maquinaria en Tiempo Real.
Una base SCADA/IoT portátil y escalable.

Construida sobre herramientas open source, esta capa proporciona una agnóstica y horizontalmente escalable para monitoreo, control e integración de datos – desde una sola máquina hasta clústeres globales.

01. Núcleo en Tiempo Real y Persistencia

MongoDB como el respaldo de datos unificado.

Base de Datos en Tiempo Real Orientada a Documentos

La plataforma aprovecha MongoDB como fuente de verdad – actuando como núcleo en tiempo real, almacén de configuración persistente e historial de Secuencia de Eventos (SOE). El modelo de datos NoSQL sin esquema permite etiquetas, servidores y usuarios ilimitados sin esquemas predefinidos, facilitando una adaptación rápida a los requisitos cambiantes del negocio.

Flujos de Cambio y Eventos Asíncronos

Usando los Flujos de Cambio de MongoDB, el sistema procesa eventos de la base de datos en tiempo real. Cada inserción, actualización o eliminación de datos desencadena reacciones asíncronas inmediatas – desde la generación de alarmas hasta el registro histórico – sin sondeo, garantizando una latencia submilisegundo incluso bajo alto rendimiento.

02. HMI Web Nativa

Visualización accesible desde cualquier dispositivo.

Interfaz Responsiva HTML5

La interfaz hombre-máquina está construida enteramente sobre HTML5, JavaScript y SVG. Funciona de forma nativa en cualquier navegador moderno sin add-ons.

Editor SVG Basado en Inkscape

Los gráficos de proceso dinámicos se crean usando Inkscape, un editor de gráficos vectoriales profesional open source. Los usuarios pueden diseñar SVGs, vincularlos a puntos de datos sin necesidad de software de propietario.

Gestor de Configuración Basado en Web

Todos los parámetros del sistema – etiquetas, controladores, alarmas, usuarios – se configuran a través de una interfaz web segura. Los cambios se aplican en vivo, sin reiniciar servicios, permitiendo una adaptación dinámica a modificaciones del proceso.

Control de Acceso Basado en Roles (RBAC)

Permisos detallados controlan quién puede ver, editar o confirmar alarmas. RBAC se integra con LDAP/Active Directory existente o funciona como un sistema de autenticación independiente, garantizando la seguridad operacional.

03. Protocolos y Arquitectura EDGE

Conectividad ilimitada, escalable horizontalmente.

  • Amplia Biblioteca de Controladores – Soporte nativo para IEC 60870‑5‑104, DNP3, Modbus (TCP/RTU), MQTT, OPC UA, Siemens S7, BACnet, IEC 61850 MMS/GOOSE, y más. Se pueden agregar nuevos controladores en cualquier lenguaje (Node.js, C#, Golang) leyendo y escribiendo en colecciones de MongoDB.
  • Nodos de Adquisición de Datos Redundantes y Livianos – Colectores distribuidos se conectan de forma segura a través de TLS a la base de datos central. Cada nodo puede manejar múltiples protocolos, y los nodos se pueden agrupar en clústeres para redundancia y equilibrio de carga. La escalabilidad horizontal se logra mediante el particionamiento (sharding) de MongoDB y la orquestación de contenedores (Docker, Kubernetes).
  • Procesamiento EDGE y Concentración de Datos – Los nodos pueden realizar cálculos locales, filtrado de alarmas y agregación de datos antes de enviarlos, reduciendo el uso de ancho de banda en entornos remotos o de baja conectividad.

04. Análisis Avanzado

Historiador PostgreSQL/TimescaleDB

Los datos de series de tiempo de alta resolución se almacenan en PostgreSQL con la extensión TimescaleDB, optimizada para inserciones rápidas y consultas temporales complejas. Este historiador agrega automáticamente la telemetría sin procesar en intervalos configurables (minuto, hora, día) para el análisis de tendencias a largo plazo.

Integración Nativa con Grafana

El historiador se conecta directamente a Grafana, permitiendo la creación instantánea de paneles, alertas y consultas ad‑hoc. Los ingenieros mecánicos pueden visualizar espectros de vibración, tendencias de temperatura y degradación de la calidad del aceite sin escribir SQL. El motor de alertas de Grafana se integra con el visor de eventos para la escalada automatizada de notificaciones.

Extensibilidad para Big Data y ML

A través del conector MongoDB Spark, la plataforma se alimenta directamente a pipelines de big data y frameworks de aprendizaje automático. Los modelos predictivos para fallas de rodamientos o desalineación de transportadores pueden entrenarse con el conjunto completo de datos del historiador y desplegarse nuevamente en el borde.

Integración con Herramientas de Terceros

Hay conectores estándar disponibles para Power BI, Metabase, Node‑RED, Alerta y Kafka. La plataforma también puede servir como concentrador de datos para sistemas ERP/MES, conectando las operaciones en tiempo real con la planificación empresarial.
